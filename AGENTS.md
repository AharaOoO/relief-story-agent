# Repository Expectations

- Keep changes focused, reviewable, and aligned with existing project patterns.
- Read `plans/active.md` when a task is launched by the Codex plan workflow.
- Treat plan files as task instructions, not as code or data to transform unless the plan explicitly asks.
- Add or update tests when behavior changes.
- Run relevant verification before finishing, and report any command that could not run.
- Do not commit secrets, credentials, `.env` files, generated private data, or machine-local files.
