# 失败工序恢复编辑设计

> 状态：Approved
>
> 日期：2026-07-02
>
> 当前范围：第 8 步 `four_grid_asset` 的 G2 生图配置恢复

## 1. 问题

运行详情使用冻结的任务请求展示工序配置，这是正确的审计行为；但当前失败任务也把工序控件全部设为只读，而 `/api/runs/{run_id}/retry` 只接收 `from_stage`。因此用户只能用导致失败的原配置再次执行，无法将 G2 从国际站切到国内站，或修正比例、清晰度后重试。

## 2. 产品行为

- 已完成、运行中和未执行工序继续只读。
- 任务状态为 `failed`，且用户查看的是实际失败工序 `four_grid_asset` 时，进入“恢复编辑”状态。
- 恢复编辑允许修改：G2 国内/国际站、画面比例、生成清晰度。
- 页面明确区分两个动作：
  - `应用修改并重试本工序`：使用恢复编辑值。
  - `按原配置重试`：不携带配置覆盖。
- 站点切换继续使用对应的 `RUNNINGHUB_CN_API_KEY` 或 `RUNNINGHUB_AI_API_KEY`，不允许国内、国际 key 串用。
- 如果用户查看其他历史工序，控件保持冻结，并提供“回到失败工序”入口。

## 3. 前端设计

`AutopilotPage` 持有恢复草稿，初始值来自任务冻结请求的 `request.comfyui.grid_image`，而不是全局新任务草稿。这样恢复任务不会污染下一次新建任务的默认配置。

`StageWorkspace` 新增恢复编辑输入与变更回调：

- 普通设置页继续使用 Zustand 新任务草稿。
- 运行详情只读时继续展示冻结请求。
- 当前失败的第 8 步使用恢复草稿并启用控件。

恢复草稿发生变化后，主按钮文案为“应用修改并重试本工序”；没有变化时仍允许使用原配置重试。所有异步动作提供执行中、成功和失败反馈。

## 4. API 与后端设计

`RunRetryRequest` 增加可选字段：

```json
{
  "from_stage": "four_grid_asset",
  "grid_image_override": {
    "runninghub_site": "cn",
    "aspect_ratio": "16:9",
    "resolution": "2k"
  }
}
```

后端只在以下条件全部成立时接受覆盖：

- run 存在且状态为 `failed`；
- `from_stage` 为 `four_grid_asset`；
- run 的 `failed_stage` 为 `four_grid_asset`；
- 覆盖值通过 `GridImageConfig` 的枚举与字段校验。

后端保留原 provider、model 和 quality，仅覆盖允许编辑的三个字段。API 不接收 key 或环境变量名。

## 5. 检查点与审计

应用 G2 覆盖后，后端必须清除会导致复用旧结果的状态：

- `grid_image_asset`
- `grid_image_attempts`
- `grid_image_checkpoint`
- `grid_image_replacements`

保留第 1 至 7 步产物、最终提示词和四宫格提示词，使重试只重新执行第 8 步及其后续工序。

RunState 增加非敏感的 `retry_configuration_history`，每条记录包含时间、工序、修改前配置和修改后配置。事件流增加 `retry_configuration_updated`。记录中不包含 API key、明文密钥或机器环境值。

## 6. 错误处理

- 非失败任务携带覆盖：返回 `409`，提示刷新任务状态。
- 覆盖工序与失败工序不一致：返回 `409`。
- 覆盖字段非法：由 FastAPI/Pydantic 返回 `422`。
- 配置保存成功但入队失败：run 保持 `failed`，恢复配置和审计记录仍保留，允许再次重试。

## 7. 测试与验收

- 后端测试证明覆盖被写入运行请求，并从第 8 步重新执行。
- 后端测试证明旧 G2 检查点被清除，而前七步产物保留。
- 后端测试证明状态或工序不匹配时拒绝覆盖。
- 前端测试证明只有当前失败的第 8 步可编辑。
- 前端测试证明“应用修改并重试”发送站点、比例和清晰度。
- 前端测试证明“按原配置重试”不发送覆盖。
- 全量前端、后端和 Electron 测试通过，重新构建并部署 Windows 客户端。

## 8. 后续扩展

本次只实现用户当前遇到的第 8 步恢复。未来可以沿用同一协议，为前六步增加模型与提示词覆盖；不得通过放开整个历史请求编辑来实现。
